class lumos.model.asacc.ASAcc(acc_id, ker_obj, area, tech, tech_model)[source]¶

Bases: object

Model an application-specific hardware accelerator, or ASIC accelerator.

An accelerator has the following properties:

acc_id
Accelerator identifier, this is in line with the acc_id in kernel description. This property is used to model alternate accelerators for the same computing kernel
ker_obj
The kernel object, for which the accelerator is designed.
area
The area of the accelerator, a metric of allocated resources.
tech
The technology nodes, e.g. 22(nm).
tech_model
The technology model, it could be CMOSTechModel or TFETTechModel.
vdd
The working supply voltage, it is initialized to the nominal supply of the associated technology node.

Create kernels with relative U-core performance following a normal distribution. The actual sampled kernels have the relative performance uniformly drawn from the range of 0.5*mean to 1.5*mean, if not specified. The probability of presence in an application for a kernel is the CDF of its relative performance in the normal distribution.

Args:
dist_params (dict):
Distribution parameters, such as mean, standard deviation.
fname (str):
file to save the generated kernels, default to fixednorm.xml
size (int):
number of kernels to create, default to 100.
kid_prefix (str):
prefix string for kernel id
perf_range (pair):
A pair of (min,max) specifying the performance range, if not set, default to 0.5*mean as min, and 1.5*mean as max.
Returns:
N/A

Build a workload composed of applications with the specified kernels coverage following a specified distribution.

Args:
app_num (int):
The number of applications in a workload.
kernels (list):
The list of kernel IDs.
cov_dist (str):
The distribution type of coverage, currently supports normal, lognormal, uniform.
cov_param1,cov_param2 (number):
The statistical parameters for the distribution. Their real meanings depend on the specific distribution. For normal and lognormal, param1 is the mean, and param2 is the standard deviation. For uniform, param1 is the minimum range, and param2 is the maximum range.
Returns:
workload (list):
A list of successfully built applications.

class lumos.model.Budget(area=0, power=0, bw=None)[source]¶

Bases: object

Physical constraints for a system, including area, power, and memory bandwidth

area¶

Total area of a system, in mili-meters (mm^2)

bw¶

Total available memory bandwidth of a system, in GBytes/S.

bw is in a form of python dict indexed by technology nodes, which looks like:

{45: 180, 32: 198,
 22: 234, 16: 252}
power¶

Total TDP of a system, in watts (W)

lumos.model.Sys_L = <lumos.model.Budget object>¶

A predefined budget for large system, similar to SPARC T4

lumos.model.Sys_M = <lumos.model.Budget object>¶

A predefined budget for medium system, similar to Xeon processor

lumos.model.Sys_S = <lumos.model.Budget object>¶

A predefined budget for small system, similar to Core2 processor
